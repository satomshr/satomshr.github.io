Kaggle の Titanic のチュートリアルをトライ

検討結果

Kaggle の Titanic に 取り組み始めました. まずはチュートリアル通りにコードを書きました. 結果は以下の通り.

  • gender_submission.csv (女性は生存, 男性は死亡) ; 0.76555
  • my_submission.csv (random forest) ; 0.77511

チュートリアルで考慮しているのは Pclass (チケットクラス), Sex (性別), SibSp (同乗している兄弟, 配偶者の数), Parch (同乗している親, 子供の数) の 4 つだった. そこで Age (年齢) を加えれば精度が上がるかな? と思ってコードを修正した. ただし, Age には N/A が含まれているので, codexa の解説 を 参考にして, Age の N/A を median (中央値) で埋めた. 結果は…

  • Pclass, Sex, SibSp, Parch + Age (random forest) ; 0.77272

Before が 0.77511, After が 0.77272 で, その差は 0.003 だが下がっている. 世の中, そう簡単にはうまくいかないのだ :-p
